# Block: Site Header (Dynamic)
**Block name:** `kw-package/site-header`
**Version:** 2.0.0
**Generated:** 2026-04-05
**Pencil source:** `designs/design-components-blocks.pen` (Nav 1-5 + hero navbars)

## Purpose
Dynamic FSE site header block that renders server-side via `render.php`. Navigation items come from WordPress native menus (Appearance > Menus), supporting custom URLs, open-in-new-tab, CSS classes, and descriptions out of the box. Five style variants cover standard SaaS, transparent overlay, search/marketplace, mega menu enterprise, and e-commerce retail layouts. Each variant supports light and dark color modes, responsive mobile hamburger menu, and sticky positioning.

## Architecture
This is a **dynamic block** -- `save.js` returns `null` and all frontend HTML is generated by `render.php` using `wp_nav_menu()`. The editor uses `ServerSideRender` to preview the PHP output in real-time. Navigation links are no longer stored as block attributes; they come from registered WordPress menu locations.

## Frontend Behaviour
- **Mobile menu:** Hamburger button toggles a full-screen overlay menu with slide-in animation. Close button and Escape key dismiss it. Body scroll is locked while open. Focus is trapped inside the menu.
- **Mega menu (mega variant):** Clicking the first nav link (Solutions) toggles a dropdown panel with animated slide-down. Clicking outside or pressing Escape closes it. Chevron icon rotates 180 degrees when expanded.
- **Search toggle (ecommerce variant):** Search icon toggles the search bar visibility.

## Known Constraints
- Navigation items must be assigned via Appearance > Menus in wp-admin. If no menu is assigned, no links render.
- The mega menu variant requires menus assigned to "Mega Menu 1" and "Mega Menu 2" locations for the two dropdown columns.
- Menu item descriptions (for mega menu items) require enabling "Description" in Screen Options on the Menus admin page.
- Adding a new style variant requires updating the `$structures` and `$colors` arrays in render.php, and the `STYLE_STRUCTURE` object in edit.js.
- The e-commerce cart badge count is static (shows "0"). For WooCommerce integration, update render.php to use `WC()->cart->get_cart_contents_count()`.
